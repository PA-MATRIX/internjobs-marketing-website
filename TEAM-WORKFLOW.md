# Team workflow — two developers, RRR, integration branch + CI

This repo is built by two developers running RRR team mode:

| Team | Owner | Owns | Branch prefix |
|------|-------|------|---------------|
| `team-cms` | Raj (coordinator) | Marketing CMS · Student app · Startups/Employers | `rrr/<ver>/team-cms[-NN]` |
| `team-workspace` | Nithin | Workspace (apps/parrot) | `rrr/<ver>/team-workspace[-NN]` |

## Branch model

```
main                      production · protected · coordinator-only merge · deploys (manual)
└── integration/<ver>     shared staging · both devs PR here · CI runs the full suite
    ├── rrr/<ver>/team-cms[-NN]        (Raj)
    └── rrr/<ver>/team-workspace[-NN]  (Nithin)
```

- A phase branch opens its PR against **`integration/<ver>`**, not `main`.
- CI runs on that PR (typecheck + build + unit tests). Green is required to merge.
- When the milestone is complete and `integration/<ver>` is green, the coordinator
  opens **one** PR `integration/<ver> → main`, merges it, and deploys.
- This is the only place the two teams' work is built and tested *together* before
  it becomes permanent on `main`.

## RRR state files are NOT merged

RRR's rolling state files are **local execution memory** — they are rewritten every
phase and are regenerated by RRR. They are git-ignored and must never be committed,
because every phase branch would otherwise drag a fresh copy to `main` and collide
with the previous one (the recurring "STATE.md conflict").

Git-ignored (see `.gitignore`):

- `.planning/STATE.md`              (coordinator working state — local only)
- `.planning/workstreams/*/STATE.md` (per-team execution memory — local only)
- `.planning/learnings.jsonl`        (append-only telemetry — local only)

**Still tracked and reviewed** (durable, per-phase, do not collide):
`PLAN.md`, `SUMMARY.md`, `VERIFICATION.md`, `ROADMAP.md`, `REQUIREMENTS.md`.
`ROADMAP.md` / `REQUIREMENTS.md` are coordinator-owned — only edit them on `main`.

### One-time migration step (each developer, once)

These files used to be tracked. After pulling the commit that ignores them, your
local copy stays on disk but git stops tracking it. If git tries to delete your
working copy on pull, stash first:

```bash
git stash push -- .planning/STATE.md ".planning/workstreams/*/STATE.md" .planning/learnings.jsonl
git pull
git stash pop   # restores your local state files (now untracked)
```

RRR will regenerate any of these it needs.

## CI

`.github/workflows/ci.yml` runs on PRs to `main`/`integration/**` and on pushes to
`integration/**`:

- **workspaces** — `npm ci`; build marketing, employers, and verify the student app.
- **parrot** — `npm ci`; `npm run typecheck`; `npm run build`.
- **startup** — `npm ci`; `npm run typecheck`; `npm test` (node:test via `tsx`).

A `wrangler deploy --dry-run` runs on integration pushes as an **advisory** (non-blocking)
config-contract check. Deploys remain manual — no Cloudflare credentials live in
GitHub Actions yet. To enable real CD later, add `CLOUDFLARE_API_TOKEN` +
`CLOUDFLARE_ACCOUNT_ID` as Actions secrets and add a `deploy.yml` gated on `main`.

## Day-to-day

```bash
# start a phase off the integration branch
git fetch origin
git switch -c rrr/v1.4/team-cms-30 origin/integration/v1.4

# ...do the work, commit...

# open the PR against integration (NOT main)
gh pr create --base integration/v1.4 --fill
```
