Create `apps/parrot/app/routes/admin.tsx` as a React client component (no server loader — the parrot app is a Hono/React SPA, not a Remix app; all data is fetched client-side via `fetch` + `useEffect`).

The component renders:
1. A page header ("Admin" title + "Invite new employee" link pointing to `/admin/invite`).
2. An employee table loaded from `GET /api/admin/employees`. Columns: Display name, Workspace email, Status, Capabilities, Actions.
3. For "Capabilities" column: show 6 pill badges (email / chat / meetings / phone / sms / campaigns). Each pill is green when the flag is `true`, gray when `false`. Flags are loaded per-employee from `GET /api/admin/employees/:id/flags`.
4. For "Actions" column: an "Edit" button per row. Clicking it expands an inline capability editor (NOT a modal — just a row below the employee row). The editor shows 6 labeled checkboxes (one per capability). Submitting the editor calls `PATCH /api/admin/employees/:id/flags` with `{ featureFlags: {...} }` and then reloads the flags for that row.
5. A "Disable" button that calls `DELETE /api/admin/employees/:id`. Disabled employees show a muted row.

TypeScript interfaces needed (inline in this file, don't add to shared types):

```ts
interface EmployeeRow {
  id: string;
  clerk_user_id: string;
  workspace_email: string;
  personal_email: string;
  display_name: string;
  status: "invited" | "active" | "disabled";
  created_at: string;
}

type CapabilityKey = "email" | "chat" | "meetings" | "phone" | "sms" | "campaigns";

interface CapabilityFlags {
  email: boolean;
  chat: boolean;
  meetings: boolean;
  phone: boolean;
  sms: boolean;
  campaigns: boolean;
}
```

State shape:
- `employees: EmployeeRow[]` — loaded on mount
- `flags: Record<string, CapabilityFlags>` — keyed by employee.id, loaded lazily (fetch for each row after employee list loads)
- `editingId: string | null` — which row is in edit mode
- `editDraft: CapabilityFlags | null` — the in-progress edit values
- `busy: Record<string, boolean>` — per-row loading state
- `pageError: string | null`

Styling: use existing Tailwind classes matching `admin.invite.tsx`'s style (slate-900 background for header elements, slate-200 borders, sm rounded tables). The pill badges use `bg-emerald-100 text-emerald-800` for active and `bg-slate-100 text-slate-500` for inactive.
