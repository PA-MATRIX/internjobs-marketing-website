<objective>
Implement the `closeTodoFact` Cypher writer so that Phase 19's auto-clear cron becomes functional.

Purpose: Phase 19 shipped the `runAutoClear` cron (apps/parrot/workers/lib/auto-clear.ts) which queries FalkorDB for :Todo nodes where valid_to IS NOT NULL and closes the linked SQLite todos. But the writer that SETS valid_to was never implemented тАФ `closeTodoFact` does not exist in graph.ts. Every resolution-acknowledgement reply leaves valid_to as null, so the cron query returns zero rows and the Resolved view stays empty forever.

This plan adds the missing writer in three places:
1. `infra/graph-api/src/index.mjs` тАФ new POST /close-todo endpoint that executes the SET Cypher
2. `apps/parrot/workers/lib/graph.ts` тАФ new exported `closeTodoFact()` function that calls the graph-api proxy
3. `apps/parrot/workers/routes/reply-forward.ts` тАФ reply handler invokes `closeTodoFact` when the sent reply body matches a resolution-acknowledgement regex

Output: :Todo.valid_to written тЖТ cron ticks within 30s тЖТ DO.resolveTodo flips resolved_at тЖТ todo appears in Resolved view.
</objective>

    Add a new POST /close-todo endpoint to `infra/graph-api/src/index.mjs` immediately after the existing POST /query handler, following the exact same auth + error-handling pattern.

    The endpoint:
    - Accepts JSON body: `{ thread_id: string, employee_id: string, resolution_text?: string }`
    - Validates required fields (thread_id + employee_id); returns 400 `{ error: "thread_id_and_employee_id_required" }` if missing
    - Requires Bearer auth via the existing `verifyBearer()` helper; returns 401 if missing/invalid
    - Executes this Cypher:
      ```
      MATCH (t:Todo)
      WHERE t.source_id = $thread_id AND t.employee_id = $employee_id AND t.valid_to IS NULL
      SET t.valid_to = timestamp()
      RETURN count(t) AS closed_count
      ```
    - Returns `{ ok: true, closed_count: <number> }` on success
    - Returns 503 if FalkorDB client is unavailable
    - Returns 500 on query error with `{ error: "close_failed", detail: err.message }`

    IMPORTANT: The Cypher uses `t.source_id = $thread_id` тАФ in the :Todo schema, source_id is the email message id or chat post id that seeded the todo. The caller passes the thread_id (which maps to source_id in the graph namespace). If multiple :Todo nodes share the same source_id + employee_id (unlikely but possible on re-runs), SET updates all of them тАФ that is correct behavior (all todos from the same source are resolved together).

    CONVENTIONS from existing code:
    - Use the same `verifyBearer(req)` function already defined in the file
    - Use the same `getClient()` singleton тАФ do NOT add a new DB client
    - Log query failures via the same `console.warn(JSON.stringify({...}))` pattern
    - The endpoint name is `/close-todo` (kebab-case, matches the /query naming convention)

    PART A тАФ graph.ts:

    Add a new exported function `closeTodoFact` to the end of `apps/parrot/workers/lib/graph.ts` (after the existing `pingParrotGraph` function). Follow the exact same fail-soft pattern used by all other exported functions in this file.

    ```typescript
    export interface CloseTodoFactArgs {
      /** Source id of the :Todo to close (matches t.source_id in FalkorDB). */
      threadId: string;
      /** Clerk user id of the owning employee (matches t.employee_id in FalkorDB). */
      employeeId: string;
      /** The acknowledgement phrase that triggered the close (for structured log). */
      resolutionText?: string;
    }

    export interface CloseTodoFactResult {
      /** Number of :Todo nodes that had valid_to set (0 = already closed or not found). */
      closedCount: number;
    }

    /**
     * Sets valid_to = timestamp() on the matching :Todo node via the
     * graph-api POST /close-todo endpoint. Called from the reply handler
     * when the agent reply body matches a resolution-acknowledgement phrase.
     *
     * Fail-soft: returns null on any error (proxy unavailable, network
     * error, non-2xx). The reply is still sent тАФ graph state is best-effort.
     *
     * REQUIREMENTS: CLOSETODO-01, CLOSETODO-02
     */
    export async function closeTodoFact(
      env: GraphEnv,
      args: CloseTodoFactArgs,
    ): Promise<CloseTodoFactResult | null> {
      if (!env.GRAPH_API_URL || !env.GRAPH_API_SECRET) return null;
      if (!args.threadId || !args.employeeId) return null;

      try {
        const res = await fetch(
          env.GRAPH_API_URL.replace(/\/$/, "") + "/close-todo",
          {
            method: "POST",
            headers: {
              "Content-Type": "application/json",
              Authorization: `Bearer ${env.GRAPH_API_SECRET}`,
            },
            body: JSON.stringify({
              thread_id: args.threadId,
              employee_id: args.employeeId,
              resolution_text: args.resolutionText ?? "",
            }),
            signal: AbortSignal.timeout(5000),
          },
        );
        if (!res.ok) {
          console.warn(
            JSON.stringify({
              level: "warn",
              message: "parrot_graph_close_todo_error",
              status: res.status,
              threadId: args.threadId,
            }),
          );
          return null;
        }
        const body = (await res.json()) as { ok?: boolean; closed_count?: number };
        const closedCount = Number(body.closed_count ?? 0);
        // Structured log тАФ CLOSETODO-04
        console.log(
          JSON.stringify({
            level: "info",
            event: "todo_fact_closed",
            thread_id: args.threadId,
            employee_id: args.employeeId,
            matched_phrase: args.resolutionText ?? "",
            closed_count: closedCount,
          }),
        );
        return { closedCount };
      } catch (err) {
        console.warn(
          JSON.stringify({
            level: "warn",
            message: "parrot_graph_close_todo_fetch_failed",
            error: (err as Error)?.message ?? String(err),
            threadId: args.threadId,
          }),
        );
        return null;
      }
    }
    ```
