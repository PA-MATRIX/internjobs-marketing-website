    This is a new file. Follow the exact same pattern as `apps/parrot/workers/routes/reply-forward.ts` — use Hono with a `ParrotContext` type, access the authenticated employee via `c.var.employee`, and use `c.var.mailboxStub` to verify ownership before hitting R2.

    ```typescript
    // v1.4 Phase 23 ATTACH-DOWN-01..03: Attachment download route.
    //
    // GET /api/inbox/messages/:messageId/attachments/:attachmentId
    //
    // Returns the R2 blob with correct Content-Type + Content-Disposition.
    // Auth: Clerk session (handled by requireEmployeeMailbox middleware in index.ts).
    // Ownership: The route verifies that the :messageId belongs to the authenticated
    //   employee's mailbox before fetching from R2. Non-owner → 403. Missing → 404.
    //
    // R2 key convention: attachments/{clerk_user_id}/{messageId}/{attachmentId}/{filename}
    //   Written by apps/parrot/workers/lib/inbound-email.ts. We reconstruct the key
    //   by: (1) verifying the email belongs to the employee's DO, (2) looking up the
    //   attachment metadata from the DO to get filename + mimetype, (3) fetching the R2
    //   object at the reconstructed key.
    //
    // ATTACH-DOWN-02: Non-owner → 403 (ownership enforced via DO lookup under the
    //   authenticated employee's clerk_user_id — a different employee's DO simply
    //   won't have the messageId row, returning null → we return 403 not 404 to
    //   avoid leaking existence of the message via timing).
    //
    // Skills: cloudflare Workers + R2 + Durable Objects

    import type { Context } from "hono";
    import type { ParrotContext } from "../lib/mailbox";

    type AppContext = Context<ParrotContext>;

    /**
     * GET /api/inbox/messages/:messageId/attachments/:attachmentId
     *
     * Returns the R2 blob as a streaming response with correct headers.
     * Requires Clerk session (enforced by requireEmployeeMailbox in index.ts).
     */
    export async function handleAttachmentDownload(c: AppContext) {
      const messageId = c.req.param("messageId") ?? "";
      const attachmentId = c.req.param("attachmentId") ?? "";

      if (!messageId || !attachmentId) {
        return c.json({ error: "missing_params" }, 400);
      }

      const employee = c.var.employee;
      const stub = c.var.mailboxStub;

      // ATTACH-DOWN-02: Verify ownership by looking up the attachment metadata
      // through the authenticated employee's DO. If the email or attachment
      // doesn't exist in their mailbox, return 403 (not 404) to avoid leaking
      // that the message exists but belongs to someone else.
      let attachmentMeta: {
        id: string;
        filename: string;
        mimetype: string;
      } | null = null;

      try {
        // getAttachment is expected to be an RPC on EmployeeMailboxDO.
        // If it doesn't exist yet, fall back to getEmail + search attachments.
        const email = await (stub as unknown as {
          getEmail(id: string): Promise<{
            id: string;
            attachments?: Array<{
              id: string;
              filename: string;
              mimetype: string;
            }>;
          } | null>;
        }).getEmail(messageId);

        if (!email) {
          // Message not found in this employee's mailbox — ownership check fails
          return c.json({ error: "forbidden" }, 403);
        }

        const att = email.attachments?.find((a) => a.id === attachmentId);
        if (!att) {
          return c.json({ error: "not_found" }, 404);
        }

        attachmentMeta = att;
      } catch (err) {
        console.warn(
          JSON.stringify({
            level: "warn",
            event: "attachment_lookup_failed",
            messageId,
            attachmentId,
            error: (err as Error)?.message ?? String(err),
          }),
        );
        return c.json({ error: "lookup_failed" }, 500);
      }

      // Reconstruct the R2 key. Convention from inbound-email.ts:
      // attachments/{clerk_user_id}/{messageId}/{attachmentId}/{filename}
      const filename = attachmentMeta.filename || "download";
      const r2Key = `attachments/${employee.clerkUserId ?? employee.email}/${messageId}/${attachmentId}/${filename}`;

      const object = await c.env.BUCKET.get(r2Key);
      if (!object) {
        // Object missing from R2 — attachment metadata exists but blob is gone
        return c.json({ error: "not_found" }, 404);
      }

      // Safe filename encoding for Content-Disposition header
      const safeFilename = encodeURIComponent(filename).replace(/%20/g, "+");

      return new Response(object.body, {
        headers: {
          "Content-Type": attachmentMeta.mimetype || "application/octet-stream",
          "Content-Disposition": `attachment; filename="${filename}"; filename*=UTF-8''${safeFilename}`,
          "Cache-Control": "private, max-age=3600",
          "Content-Length": String(object.size),
        },
      });
    }
    ```

    IMPORTANT: The `employee.clerkUserId` field name — verify the exact field by reading `apps/parrot/workers/lib/mailbox.ts`. Use whatever field holds the Clerk user_id (it may be `clerkUserId`, `userId`, or `employeeId`). The R2 key in `inbound-email.ts` uses `employee.clerk_user_id`. Match exactly.

    Also check: does `EmployeeMailboxDO` have a `getEmail` method that returns attachments? Read `apps/parrot/workers/durableObject/index.ts` briefly. If `getEmail` does NOT return attachments, look for `getAttachment(messageId, attachmentId)` or query the DO's `getEmails` with filters. The key constraint: resolve attachment metadata (filename, mimetype) through the authenticated employee's DO — do not trust client-supplied values.

    PART B — Wire into index.ts:

    In `apps/parrot/workers/index.ts`, import the handler and mount it on the Hono router INSIDE the `requireEmployeeMailbox` middleware block (so it inherits Clerk auth). Add after the existing agentRoutes mount:

    ```typescript
    api.get("/inbox/messages/:messageId/attachments/:attachmentId", handleAttachmentDownload);
    ```

    Look at how `agentRoutes` and `handleReplyEmail` are mounted to understand the correct position in the file.

    Update the attachment chip rendering to produce a direct download link:
    ```tsx
    // For each attachment, build a link to the new download route:
    const downloadUrl = `/api/inbox/messages/${email.id}/attachments/${att.id}`;

    // Render as an anchor tag that triggers browser download:
    <a
      href={downloadUrl}
      download={att.filename}
      target="_blank"
      rel="noopener noreferrer"
      className={/* existing chip classes */}
    >
      {/* existing chip content (icon + filename + size) */}
    </a>
    ```

    IMPORTANT rules:
    1. Use `email.id` (the message UUID) not `email.message_id` (the RFC-5322 header value). The route param `:messageId` maps to the DO's internal UUID (what was used as the R2 key base in inbound-email.ts: `const messageId = crypto.randomUUID()`).
    2. Use `att.id` for `:attachmentId` — this is the UUID generated at inbound time (`const attId = crypto.randomUUID()`).
    3. The `download` attribute on the anchor tag tells the browser to save the file rather than navigate. Include `att.filename` as the download attribute value.
    4. Keep `target="_blank"` so the download doesn't navigate away from the email view.
    5. Do NOT add a new `fetch` call or React Query dependency — a plain `<a href>` is correct because the Worker returns the blob directly with `Content-Disposition: attachment`.
    6. Do NOT break existing chip styling — apply the download link wrapping AROUND the existing chip JSX, not replacing the inner content.

    If the current EmailPanel doesn't render attachment chips at all (empty state), add a minimal chip renderer:
    ```tsx
    {email.attachments && email.attachments.length > 0 && (
      <div className="flex flex-wrap gap-2 mt-3 border-t pt-3">
        {email.attachments.map((att) => (
          <a
            key={att.id}
            href={`/api/inbox/messages/${email.id}/attachments/${att.id}`}
            download={att.filename}
            target="_blank"
            rel="noopener noreferrer"
            className="flex items-center gap-1.5 px-3 py-1.5 text-sm rounded-full bg-gray-100 hover:bg-gray-200 transition-colors"
          >
            <span>{att.filename}</span>
            <span className="text-gray-400 text-xs">
              {att.size ? `(${Math.round(att.size / 1024)}KB)` : ""}
            </span>
          </a>
        ))}
      </div>
    )}
    ```
