      curl -s -X POST https://app.internjobs.ai/internal/safety-events \
        -H "Authorization: Bearer $INTERNAL_API_SECRET" \

        -H "Authorization: Bearer $INTERNAL_API_SECRET"

      curl -s -o /dev/null -w "%{http_code}" -X POST \
        https://app.internjobs.ai/internal/safety-events \
        -H "Authorization: Bearer wrong-secret-value" \
        -H "Content-Type: application/json" \
        -d '{"channel":"email","action":"blocked"}'

    Student app must not crash (subsequent probe 1a with correct secret still

      `flyctl proxy 15432:5432 --app internjobs-student-db` and check connectivity
    - 500 on list: same DB issue

    Fix any issue found, redeploy if needed, and re-run the failing probe until
    it matches the expected output.

    shipped in the Neon-exit (infra/NEON-EXIT.md); this task is verification, not
    feature work.
