    **infra/startup-api/src/index.mjs** — add two new Bearer-gated endpoints:
    
    `GET /v1/touchbase/due-startups`:
    - Query: return startups that need a touchbase SMS.
    - SQL:
      ```sql
      SELECT
        cl.id          AS channel_link_id,
        cl.startup_id,
        cl.channel_external_id AS phone,
        cl.member_id,
        s.name         AS startup_name,
        m.name         AS founder_name
      FROM startup_channel_links cl
      JOIN startups s ON s.id = cl.startup_id
      JOIN startup_members m ON m.id = cl.member_id
      WHERE cl.channel_type = 'telnyx-sms'
        AND cl.status = 'active'
        AND (cl.opt_in_flags->>'weekly_touchbase')::boolean = true
        AND (cl.last_touchbase_at IS NULL
             OR cl.last_touchbase_at < NOW() - INTERVAL '7 days')
      ORDER BY cl.last_touchbase_at ASC NULLS FIRST
      LIMIT 100
      ```
    - Returns `{ due: [ { channel_link_id, startup_id, phone, member_id, startup_name, founder_name } ] }`.
    
    `PATCH /v1/channel-links/:id/touchbase-sent`:
    - Updates `last_touchbase_at = NOW()` for the given channel_link_id.
    - SQL: `UPDATE startup_channel_links SET last_touchbase_at = NOW(), updated_at = NOW() WHERE id = $1`
    - Returns 200 `{ok: true}` or 404.
    
    `GET /v1/startups/:startup_id/fresh-candidates`:
    - Returns up to 3 of the most recent active candidate threads for this startup.
    - SQL: select the 3 most recent threads (ORDER BY created_at DESC LIMIT 3) joined to roles + student_members.
    - Returns `{ candidates: [ { thread_id, candidate_name, role_title, summary } ] }`.
    - Used by scheduled.ts to build the "3 new this week" message.

    async function runWeeklyTouchbase(env: Env): Promise<void> { ... }

    `runWeeklyTouchbase` logic:
    1. `GET ${env.STARTUP_API_URL}/v1/touchbase/due-startups` with Bearer `env.STARTUP_API_SECRET`.
       On error: log and return (cron must never throw — a thrown exception fails the cron silently).
    2. For each `due` startup (up to 100 per cron run):
       a. `GET /v1/startups/${startup_id}/fresh-candidates` — get up to 3 candidates.
       b. Build message:
          ```
          hey [founder_name] — [N] new intern candidates this week for [startup_name].
          reply 1/2/3 to see a candidate, or 'stop' to opt out.
          [1. candidate_name (role_title)]
          [2. candidate_name (role_title)]
          [3. candidate_name (role_title)]
          ```
          If 0 candidates: `"hey [founder_name] — no new candidates this week, but we're actively sourcing. we'll text when someone great applies. reply 'stop' to opt out."`. Skip KV cursor write if 0 candidates.
       c. If `env.TOUCHBASE_CURSORS` is defined and there are candidates:
          Write KV cursor: key = `touchbase:cursor:${phone}`, value = JSON array of `{ thread_id, candidate_name, role_title }` for each candidate in order, TTL = 172800 (48h).
       d. `sendSms(env, phone, message)`.
       e. If SMS send succeeds: `PATCH /v1/channel-links/${channel_link_id}/touchbase-sent`.
       f. Log: `console.log('[touchbase]', { startup_id, phone: phone.slice(-4), candidate_count: candidates.length })`.
    3. Log completion: `console.log('[touchbase] cron complete', { processed: due.length })`.
    
    Guard: if `!env.TELNYX_API_KEY`: log "TELNYX_API_KEY not set — touchbase cron skipped" and return immediately (ops-deferred mode).
    Guard: if `!env.TOUCHBASE_CURSORS`: log warning but continue (cursor won't be written; reply "1/2/3" will fail gracefully in telnyx.ts since KV is unbound).

          if (entry) {
            const result = await handleExecute({
