<objective>
Wire the Telnyx Voice AI Agent's three HTTP hooks into the startup Worker:
1. Pre-call dynamic-variables hook (voice-init) — looks up caller phone to inject per-startup MCP token
2. Post-call insights hook (voice-postprocess) — fetches recording + transcript, stores to R2, writes audit log
3. Webhook-tool fallback (voice-tool) — handles the same execute() calls as MCP but via Telnyx's webhook-tool protocol, gated behind TELNYX_USE_MCP_INTEGRATION env var

Also creates `docs/VOICE_AGENT_CONFIG.md` — the copy-paste config Ridhi pastes into the Telnyx portal to configure the Voice AI Agent. All code runs without Telnyx credentials (ops-deferred guards throughout).

Purpose: Parallel with 29-03 (different files — voice.ts + lib/voice-onboarding.ts vs scheduled.ts + intent updates). Both depend on 29-01's lib files.
Output: Three voice webhook handlers; R2 wiring; VOICE_AGENT_CONFIG.md with full system prompt.
</objective>

    **lib/voice-onboarding.ts** — export `handleRegisterStartupFromVoice()`:
    
    ```ts
    export interface VoiceRegistrationArgs {
      company: string;
      founder_name: string;
      founder_email: string;
      what_hiring_for: string;
      caller_phone: string;   // from telnyx_end_user_target dynamic variable
    }
    
    export async function handleRegisterStartupFromVoice(
      env: Env,
      args: VoiceRegistrationArgs,
    ): Promise<{ ok: true; startup_id: string; mcp_install_snippet: string } | { ok: false; error: string; already_registered: boolean }>
    ```
    
    Implementation:
    1. Call `POST /admin/startups/new` on the MCP Worker's own admin endpoint via loopback
       fetch to `${env.STARTUP_API_URL}/admin/startups/new` — WAIT. The admin endpoint
       is on the MCP Worker itself (mcp.internjobs.ai/admin/startups/new), not the Fly proxy.
       Use `https://mcp.internjobs.ai/admin/startups/new` as the URL. Bearer:
       `env.STARTUP_MCP_ADMIN_SECRET`. Body: `{ company, founder_name, work_email: founder_email, what_hiring_for, phone: caller_phone, channel_type: 'telnyx-voice' }`.
    2. On 201: extract `{ startup_id, mcp_install_snippet }` from response. Return `{ ok: true, ... }`.
    3. On 409: return `{ ok: false, error: 'already_registered', already_registered: true }`.
    4. On other non-2xx: return `{ ok: false, error: 'registration_failed', already_registered: false }`.
    5. On success: `sendSms(env, caller_phone, "Your InternJobs account is ready! Install the MCP tool: ${mcp_install_snippet}. Reply YES to get weekly candidate updates.")`.
    6. Log the outcome to `writeAuditLog` with `channel: 'telnyx-voice'`, `action: 'register_startup'`,
       `startup_id: result.startup_id ?? 'onboarding'`, `member_id: 'onboarding'`, `status: result.ok ? 'ok' : 'error'`.
    
    **routes/voice.ts** — `voiceRouter` Hono sub-router:
    
    `POST /webhooks/telnyx/voice-init`:
    - Called by Telnyx before a call starts. Must respond within 1 second.
    - Extract `caller_phone` from request body (Telnyx sends: `{ data: { payload: { from: { phone_number: "..." } } } }` — same shape as SMS inbound).
    - `resolveChannelLink(env, 'telnyx-voice', caller_phone)` — for a NEW caller this returns null (first call = onboarding). For a RETURNING caller who called before, this returns their context.
    - Response:
      - If resolved (returning caller): `{ dynamic_variables: { mcp_token: "<their MCP install token>" } }`.
        To get the token, call `GET ${env.STARTUP_API_URL}/v1/startups/${ctx.startup_id}/mcp-token` — add this Fly endpoint to index.mjs (it already has the token hash but needs a raw-token lookup; skip if complex and return an empty object instead — the Voice AI agent's global TELNYX_VOICE_AGENT_TOKEN will suffice for pilot).
        **Simplification for pilot:** always return `{}` (empty dynamic variables) from this endpoint. The global TELNYX_VOICE_AGENT_TOKEN is the per-agent token; per-caller token injection is v1.5. Document this decision in a TODO comment.
      - If null (new caller): return `{}`.
    - Max response time: set a `AbortController` with 900ms timeout on any Fly API call.
    
    `POST /webhooks/telnyx/voice-postprocess`:
    - Called by Telnyx after a call ends. Payload schema (use what's confirmed from RESEARCH.md;
      be defensive with optional chaining throughout — exact field names are LOW confidence):
      ```ts
      const body = await c.req.json();
      const call_control_id: string = body?.data?.payload?.call_control_id ?? body?.call_control_id ?? 'unknown';
      const caller_phone: string = body?.data?.payload?.from?.phone_number ?? body?.caller_phone ?? '';
      const transcript: string = body?.data?.payload?.transcript ?? body?.transcript ?? '';
      const recording_url: string | null = body?.data?.payload?.recording_url ?? body?.recording_url ?? null;
      // Log the FULL raw body for the first call so we can hotfix field names
      console.log('[voice-postprocess] raw payload:', JSON.stringify(body));
      ```
    - Resolve `resolveChannelLink(env, 'telnyx-voice', caller_phone)` to get `startup_id`.
      If null: use `startup_id = 'onboarding'` (partial call during registration flow).
    - If `env.VOICE_AUDIT` is defined:
      - Store transcript: `env.VOICE_AUDIT.put(\`transcripts/${startup_id}/${call_control_id}.json\`, JSON.stringify({ transcript, call_control_id, caller_phone, timestamp: new Date().toISOString() }), { httpMetadata: { contentType: 'application/json' } })`.
      - If `recording_url`: `fetch(recording_url)` → get `audioBuffer` → `env.VOICE_AUDIT.put(\`recordings/${startup_id}/${call_control_id}.mp3\`, audioBuffer, { httpMetadata: { contentType: 'audio/mpeg' } })`.
    - If `env.VOICE_AUDIT` is undefined: log warning (R2 not yet bound — ops-deferred).
    - Partial/abandoned call detection: if `transcript` is empty or < 50 chars AND `caller_phone`:
      `sendSms(env, caller_phone, "Looks like we got cut off. Call us back to finish your InternJobs registration!")`.
    - Write audit log: `writeAuditLog({ member_id: ctx?.member_id ?? 'onboarding', startup_id: ctx?.startup_id ?? 'onboarding', channel: 'telnyx-voice', action: 'voice_call', status: 'ok', env })`.
    - Return 200 `{ok: true}`.
    
    `POST /webhooks/telnyx/voice-tool`:
    - Only active when `env.TELNYX_USE_MCP_INTEGRATION !== 'true'` (webhook-tool fallback path).
    - Telnyx sends: `{ tool_name: "...", tool_arguments: {...}, call_control_id: "...", ... }`.
    - Extract `tool_name` and `tool_arguments`. Map tool names to ACTION_HANDLERS keys (Telnyx
      tool names may differ from our action names — use a mapping table: 
      `{ register_startup: 'register_startup', show_candidate: 'show_candidate', ... }`).
    - Resolve startup context from `call_control_id` via caller phone (from tool_arguments if
      Telnyx injects `{{telnyx_end_user_target}}` into args, or from a KV cache keyed on
      `call_control_id` populated by voice-init — for pilot simplicity, extract caller phone
      from `tool_arguments.phone_number ?? tool_arguments.caller_phone`).
    - Call `handleExecute` or `handleRegisterStartupFromVoice` as appropriate.
    - Return 200 `{ result: JSON.stringify(result) }` — Telnyx async webhook tool format.
