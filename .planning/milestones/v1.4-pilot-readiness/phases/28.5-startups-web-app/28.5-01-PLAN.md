  <how-to-verify>
    Perform these steps IN ORDER. Each builds on the previous.

    **Step 1 — Create Clerk app #3**
    1. Go to https://dashboard.clerk.com → "Create application"
    2. Name: "InternJobs Startups"
    3. Enable: Google OAuth + Email magic-link
    4. Disable: Phone OTP (startup founders use work email, not phone like Workspace)
    5. Copy PublishableKey (pk_live_...) and SecretKey (sk_live_...)
    6. Run:
       ```
       wrangler secret put STARTUPS_CLERK_SECRET_KEY --config apps/startup/wrangler.jsonc
       ```
       Paste the SecretKey when prompted.
    7. Save BOTH values to Infisical:
       - Path: /internjobs-ai, env: prod
       - Key: STARTUPS_NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY = pk_live_...
       - Key: STARTUPS_CLERK_SECRET_KEY = sk_live_...
    8. From the Clerk app settings, also copy:
       - Issuer URL (looks like https://clerk.startups.internjobs.ai after domain is connected, or https://<frontend-api-host> before)
       - JWKS URL (Issuer URL + /.well-known/jwks.json)
       Save these to Infisical as STARTUPS_CLERK_ISSUER and STARTUPS_CLERK_JWKS_URL.

    **Step 2 — Clerk domain (clerk.startups.internjobs.ai)**
    1. In Clerk Dashboard → your new app → Domains → Add domain: startups.internjobs.ai
    2. Clerk will display the CNAME target (typically frontend-api.clerk.services or similar)
    3. In Cloudflare DNS for internjobs.ai zone:
       Add CNAME: clerk.startups.internjobs.ai → [Clerk-provided target] (proxy OFF for Clerk domains)
    4. Wait for Clerk dashboard to show domain as verified (usually < 5 min)

    **Step 3 — CF Pages project + custom domain**
    1. Run:
       ```
       wrangler pages project create internjobs-startups --production-branch main
       ```
    2. In Cloudflare Dashboard → Workers & Pages → internjobs-startups → Custom Domains
       → Add: startups.internjobs.ai
       (Cloudflare auto-creates CNAME + provisions TLS certificate)
    3. Verify TLS provisioned: `curl -I https://startups.internjobs.ai` should return HTTP (may be 404 until 28.5-02 deploys — that is fine; TLS must be valid)

    **Step 4 — CF Email Routing domain verification**
    1. In Cloudflare Dashboard → [internjobs.ai zone] → Email → Email Routing
    2. Click "Add address or domain" → add domain: startups.internjobs.ai
    3. Follow the DNS checklist that appears:
       - SPF: Cloudflare may auto-add `v=spf1 include:_spf.mx.cloudflare.net ~all` as a TXT record on startups.internjobs.ai
       - DKIM: Add the 2 CNAME records Cloudflare shows (they point to CF's signing infrastructure)
       - DMARC: Manually add TXT record:
         Name: _dmarc.startups.internjobs.ai
         Value: v=DMARC1; p=none; rua=mailto:dmarc@startups.internjobs.ai
    4. Wait for green checkmarks on all DNS records in the CF Email Routing checklist
    5. Domain status should show "Verified"

    **Step 5 — Catch-all routing rule (after 28.5-04 deploys — defer this sub-step)**
    NOTE: The internjobs-startups Worker does not exist yet (built in 28.5-04). SKIP this sub-step
    now and return after 28.5-04 completes:
    CF Dashboard → Email → Routing → Catch-all action → "Send to a Worker" → internjobs-startups

    **Step 6 — Clerk webhook secret (after 28.5-02 deploys — defer this sub-step)**
    NOTE: The webhook endpoint URL does not exist yet (built in 28.5-05). SKIP this sub-step
    now and return after 28.5-05 completes:
    Clerk Dashboard → Webhooks → Add endpoint → URL: https://mcp.internjobs.ai/webhooks/clerk
    Event: user.created → copy signing secret → Infisical /internjobs-ai STARTUPS_CLERK_WEBHOOK_SECRET

    **Step 7 — DNS propagation check**
    Run these checks and confirm all pass:
    ```bash
    dig CNAME clerk.startups.internjobs.ai
    dig CNAME startups.internjobs.ai
    dig TXT _dmarc.startups.internjobs.ai
    ```
    Or use https://dnschecker.org for each record.
