  <name>Task 1: Typed API client with Clerk JWT forwarding</name>
  <files>
    apps/startups/src/lib/api.ts
  </files>
  <action>
    Create apps/startups/src/lib/api.ts — a thin typed client over the /api/* proxy.

    Pattern: every function accepts a Clerk token string (from useAuth().getToken()) and passes it as
    Authorization: Bearer in the fetch call to /api/*. The Pages Function proxy receives it and forwards
    it to the Fly proxy which validates it against STARTUPS_CLERK_JWKS_URL to resolve startup_id.

    Export these functions:

    ```typescript
    // getMe: resolves startup_id, startup_name, agent_email, role_count for the authenticated founder
    export async function getMe(token: string): Promise<{
      startup_id: string;
      startup_name: string;
      agent_email: string;
      role_count: number;
    }>

    // getRoles: list roles for the authenticated startup
    export async function getRoles(token: string): Promise<Array<{
      id: string;
      title: string;
      description: string;
      created_at: string;
    }>>

    // createRole: POST a new role. Fields match the MCP execute('post_role') schema exactly.
    // Required fields: title (string), description (string)
    // Optional: location (string), employment_type ('full_time'|'part_time'|'contract'|'internship')
    export async function createRole(token: string, body: {
      title: string;
      description: string;
      location?: string;
      employment_type?: string;
    }): Promise<{ id: string; title: string }>

    // getThread: get message history for a specific thread
    export async function getThread(token: string, threadId: string): Promise<{
      thread_id: string;
      candidate_name: string;
      messages: Array<{
        id: string;
        direction: 'inbound' | 'outbound';
        body: string;
        created_at: string;
        channel: string;
      }>;
    }>

    // sendReply: post a reply to a candidate thread (outbound from agent email)
    export async function sendReply(token: string, threadId: string, body: string): Promise<{ ok: boolean }>

    // getThreads: list recent candidate threads for the startup dashboard
    export async function getThreads(token: string): Promise<Array<{
      thread_id: string;
      candidate_name: string;
      last_message_at: string;
      unread_count: number;
    }>>
    ```

    Implementation: each function does a typed fetch to /api/[resource] with Authorization: Bearer token.
    On non-2xx, throw an Error with the response status + body included in the message.
    Add a shared apiRequest() helper to avoid duplication.

    IMPORTANT: The /api/me endpoint resolves startup_id from the Clerk JWT publicMetadata (set when the
    startup is created via admin endpoint). If publicMetadata.startup_id is missing, the Fly proxy will
    return 404 — the dashboard must handle this case with a clear message: "your account isn't linked
    to a startup yet — contact hello@internjobs.ai".
  </action>
  <verify>cd apps/startups && npm run build 2>&1 | tail -3</verify>
  <done>Build passes; api.ts exports all 6 functions with correct TypeScript signatures</done>
</task>
