### Task 1 — Webhook handler + blocklist + CTA flip (commit `cc0fe9a`)

**File:** `apps/startup/workers/routes/webhooks.ts` (new, 233 LOC)

`POST /webhooks/clerk` handler with the following flow:

1. **Guard:** missing `STARTUPS_CLERK_WEBHOOK_SECRET` → log + 503. Returning
   503 (not 200) keeps Clerk's retry queue active until the secret is bound.
2. **Raw-body read:** `await req.text()` BEFORE any parse — Svix verifies
   the raw bytes.
3. **Svix verification:** `new Webhook(secret).verify(body, headers)` —
   throws on tampered body, missing headers, expired timestamp, wrong sig.
   Wrapped in try/catch → 400 on any failure.
4. **Type filter:** non-`user.created` events → 200 no-op (don't retry).
5. **Primary email extraction:** `extractPrimaryEmail()` prefers
   `primary_email_address_id` lookup, falls back to `email_addresses[0]`.
   Returns null when array is empty (OAuth-race case).
6. **OAuth-race guard:** if email is null, log + 200 no-op (don't DELETE
   a user we can't classify).
7. **Blocklist check:** `isPersonalEmail(email)` matches a `BLOCKED_DOMAINS`
   Set (~30 domains) + `gmx.*` family. Case-insensitive on the domain;
   exact-domain match (subdomains like `alice@mail.acme.com` pass through).
8. **Clerk DELETE:** if personal, `fetch('https://api.clerk.com/v1/users/:id',
   { method: 'DELETE', Authorization: Bearer STARTUPS_CLERK_SECRET_KEY })`.
   AWAITED before returning 200 so failure visibility happens immediately.
9. **Audit log:** structured JSON `{ event: "work_email_enforcement",
   action: "deleted" | "delete_failed", user_id, email_domain }` for ops
   review.

**File:** `apps/startup/workers/routes/webhooks.test.ts` (new, 319 LOC)

26 unit tests via Node 22's `node:test` (same pattern as `slug.test.ts`
from 28.5-04). Run with `npx tsx --test workers/routes/webhooks.test.ts`.

- **`isPersonalEmail()`** (15 cases): gmail/yahoo/hotmail/outlook/icloud/
  proton.me blocked; acme.io/stripe.com accepted; uppercase blocked
  (`alice@GMAIL.COM` → true); gmx.de + gmx.com blocked via family match;
  subdomain (`alice@mail.acme.com`) accepted; malformed input
  (`notanemail`) returns false; empty string returns false; trailing
  whitespace on domain is trimmed.
- **`extractPrimaryEmail()`** (4 cases): uses `primary_email_address_id`
  when present; falls back to `email_addresses[0]`; returns null on
  empty array; returns null on missing `data`.
- **`handleClerkWebhook()` integration** (7 cases via globalThis.fetch
  reassignment in try/finally): missing secret → 503; missing Svix
  headers → 400; valid sig + non-user.created → 200 no-op; valid sig +
  user.created with work email → 200 no DELETE; valid sig + user.created
  with personal email → 200 + DELETE fired against the right URL with
  `Bearer sk_test_clerk_key` header; user.created with empty
  email_addresses → 200 no DELETE (OAuth-race guard); tampered body
  with valid sig header → 400.

All 26 pass: `node --test --import tsx workers/routes/webhooks.test.ts → 26/26 pass`.
