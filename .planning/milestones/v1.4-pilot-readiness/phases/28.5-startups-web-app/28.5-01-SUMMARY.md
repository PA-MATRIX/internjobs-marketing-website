## Decisions

1. **Defer-instead-of-pause for dashboard checkpoints.** User explicitly authorized
   "don't wait on me — finish all the phases" for this session. Rather than pausing the executor
   mid-phase (which would block 28.5-02..05 from running today), captured each sub-step as a
   self-contained entry in `PHASE-28.5-DEFERRED-OPS.md` with exact acceptance criteria and a
   downstream-blocker list. The orchestrator treats Phase 28.5 as "code-complete, ops-incomplete"
   when computing pilot-readiness.

2. **JWKS_URL + ISSUER are vars, not secrets.** Both values are public Clerk endpoints — anyone with
   the Clerk app's publishable key can derive them. Declared as empty-string `vars` in
   `wrangler.jsonc` with the actual values pulled from Infisical at deploy time. This keeps
   `git diff` clean across environments while still enforcing "single source of truth = Infisical".

## Deviations (Rule 4 — Architectural, user pre-approved)

- **`PHASE-28.5-DEFERRED-OPS.md` is outside `files_modified` frontmatter.** The plan declared only
  `apps/startup/wrangler.jsonc`. The DEFERRED-OPS file is an architectural addition (the substrate
  that makes the user's "don't wait on me" instruction safe) and is required to capture the deferred
  checkpoint without losing fidelity. Documented here per HYGN-04 drift-check convention.

- **All 7 sub-steps of Task 2 are deferred.** Original `checkpoint:human-verify` design assumed
  the executor would pause and present each step interactively. User instruction overrides that
  for this session; full re-routing of the checkpoint into a written backlog is the lowest-risk
  substitute that preserves every acceptance criterion verbatim.
