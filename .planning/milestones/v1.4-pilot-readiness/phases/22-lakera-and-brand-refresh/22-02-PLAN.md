  <action>
    Before running live tests, verify the SMS insertion point wiring is in place. Read apps/app/src/server.mjs around lines 700-1100 (the SMS inbound handler). Confirm:

    1. screenMessage() is imported from ./safety/screen.mjs
    2. screenMessage() is called with (smsBody, process.env.LAKERA_GUARD_API_KEY) BEFORE runStudentInboundWorkflow
    3. If result.flagged === true AND result.score >= 0.8 (or result.action === "blocked"), the handler:
       a. Sends canned reply: "hey — couldn't process that one. try rephrasing?"
       b. Writes a safety_events row with action='blocked'
       c. Returns WITHOUT calling runStudentInboundWorkflow
    4. If result.action === 'passed_lakera_unavailable', a safety_events row is written with that action and the message proceeds
    5. If result.action === 'passed' and flagged === false (benign), NO safety_events row is written

    If the wiring is correct: proceed to Task 2.
    If any of these conditions are NOT met: this is a blocker — the v1.3 insertion points did not complete correctly. Fix the wiring in server.mjs before proceeding. Wiring pattern to follow:
    ```js
    const safety = await screenMessage(smsBody, process.env.LAKERA_GUARD_API_KEY);
    if (safety.action === 'blocked' || (safety.flagged && (safety.score ?? 0) >= 0.8)) {
      safety.action = 'blocked'; // normalize
      await writeSafetyEvent({ channel: 'sms', action: 'blocked', reason: safety.reason, score: safety.score });
      await sendReply(studentPhone, "hey — couldn't process that one. try rephrasing?");
      return; // do NOT call runStudentInboundWorkflow
    }
    if (safety.action === 'passed_lakera_unavailable') {
      await writeSafetyEvent({ channel: 'sms', action: 'passed_lakera_unavailable' });
      // fall through to agent loop
    }
    // benign (action='passed', flagged=false): no safety_events row, fall through
    ```
    Log confirmation: console.log(JSON.stringify({ level: "info", event: "lakera_screen", channel: "sms", action: safety.action }))
