  <action>
This task has NO file changes — it is a Fly secrets configuration step. Execute via Bash.

**Step 1: Set the required secret on internjobs-mattermost.**

```bash
flyctl secrets set MM_SERVICESETTINGS_ENABLEUSERACCESSTOKENS=true \
  --app internjobs-mattermost
```

This causes Mattermost to restart with the new env var. Wait for the app to restart (flyctl app status --app internjobs-mattermost) before proceeding.

**Step 2: Check for and unset the legacy stray var.**

Per research: an older env var name `ENABLEPERSONALACCESSTOKENS` may exist from prior attempts. Check:

```bash
flyctl secrets list --app internjobs-mattermost
```

If `ENABLEPERSONALACCESSTOKENS` appears in the output, unset it:

```bash
flyctl secrets unset ENABLEPERSONALACCESSTOKENS --app internjobs-mattermost
```

**Step 3: Verify PAT minting works.**

After the app restarts (~30-60s), verify that PAT minting is now enabled by calling the backfill endpoint from Wave 0:

```bash
curl -X POST https://workspace.internjobs.ai/api/admin/chat/backfill-tokens \
  -H "Authorization: Bearer <operator-session-token>"
```

Expected response: `{ "minted": N, "skipped": M, "failed": 0 }` where minted > 0 for any employees without an existing PAT.

If the response is `{ "minted": 0, "skipped": N, "failed": 0 }` it means all employees already have PATs — acceptable if PATs were minted during local testing.

**Why mmctl does NOT work for this:**

`mmctl config set ServiceSettings.EnableUserAccessTokens true` writes to MM's database-stored config but is IGNORED when a corresponding `MM_*` env var is set. The Fly deployment pins all config via env vars. See `mm-oidc-sso-blocked-by-license` memory note — same pattern confirmed. ALWAYS use `flyctl secrets set MM_SERVICESETTINGS_*` for config on this deployment.

**Note on token rotation (Wave 5 hardening candidate, per research open question 1):**

PATs can be revoked by an MM admin. Wave 0 implements 401-triggered re-mint. Periodic rotation (e.g., 90-day cron) is deferred to a future phase. Document this in the SUMMARY as a known gap.
