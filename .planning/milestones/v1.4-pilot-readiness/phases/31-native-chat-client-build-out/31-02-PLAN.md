**In workers/lib/mattermost.ts**, add the following helpers (all use the token parameter directly — callers resolve the PAT via mmFetchAsUser/resolveEmployeeToken):

- `createMmChannel(mattermostUrl, token, teamId, name, displayName, type: "O" | "P")` — POST /api/v4/channels. type "O" = public (anyone can create), type "P" = private (admin only per context decisions). Returns MattermostChannel | null.
- `joinMmChannel(mattermostUrl, token, channelId, userId)` — POST /api/v4/channels/{channelId}/members. Returns boolean.
- `editMmPost(mattermostUrl, token, postId, message)` — PUT /api/v4/posts/{postId}. Returns MattermostPost | null.
- `deleteMmPost(mattermostUrl, token, postId)` — DELETE /api/v4/posts/{postId}. Returns boolean.
- `pinMmPost(mattermostUrl, token, channelId, postId)` — POST /api/v4/channels/{channelId}/posts/pin with body { post_id: postId }. Returns boolean.
- `getMmPostThread(mattermostUrl, token, postId)` — GET /api/v4/posts/{postId}/thread. Returns MattermostPostList | null.
- `getMmTeamPublicChannels(mattermostUrl, token, teamId, page=0, perPage=100)` — GET /api/v4/teams/{teamId}/channels?page=0&per_page=100. Returns MattermostChannel[]. Use the employee's PAT (not bot) so the response reflects the user's membership state.

**In workers/index.ts**, add the following routes after the existing `/api/chat/posts` route. All use `resolveEmployeeToken(c)` from Wave 0 to get the employee PAT. All require `requireEmployeeMailbox`.

```
GET  /api/chat/channels              — list all channels for user's team (calls getMmTeamPublicChannels with user PAT)
POST /api/chat/channels              — create a new channel; body: { name, display_name, type: "O" }
POST /api/chat/channels/:id/join     — join an existing channel (user adds themselves as member)
GET  /api/chat/posts/:id/thread      — fetch full thread for a root post
POST /api/chat/posts/:id/thread      — reply to a thread; body: { message }; sets root_id = :id
PATCH /api/chat/posts/:id            — edit a message; body: { message }
DELETE /api/chat/posts/:id           — delete a message
POST /api/chat/channels/:id/pin      — pin a post; body: { post_id }
```

For POST /api/chat/channels: private channels are admin-only (gate with `hasOperatorAccess`). Public channel creation is open to all employees.

For PATCH/DELETE: verify the requesting employee is the post author before allowing edit/delete (fetch the post first; check post.user_id === employee's mm_user_id from WorkspaceDO).

All routes return appropriate errors: 503 if PAT not provisioned, 403 if unauthorized, 400 if body missing required fields.
