<objective>
Brand-refit the /admin and /admin/invite pages to use the v1.4 InternJobs brand token system.

Purpose: Both admin pages exist and are functionally correct from v1.2 Phase 16. They use Tailwind utility classes with slate/rose/emerald color names — not the brand token system (lavender/ink/cobalt/lime/tangerine/cream) introduced in Phase 22. This plan:
1. Declares brand CSS variables in apps/parrot/app/index.css (:root block)
2. Refits admin.tsx and admin.invite.tsx to use var(--lavender), var(--ink), var(--cobalt) via Tailwind arbitrary values or direct style attributes — removing raw slate/rose/emerald color references from structural surfaces (backgrounds, text, borders)
3. Updates typography: Inter font declared in index.css body rule; headlines lowercase per BRAND-V1.md voice rules

Scope boundary: "No hex literals" is the BRAND-V1.md rule. Slate/rose/emerald Tailwind classes map to hex internally — they must be replaced on branded surfaces (page background, card backgrounds, text, CTA buttons). Exception: UI state micro-colors (focus rings, disabled opacity) may keep Tailwind utility classes if they already use ink at 60/40/20% opacity semantics.

The backend API (admin-employees.ts) is NOT modified. Only frontend (.tsx files + index.css).

Reqs closed: ADMIN-UX-01 (employee directory with capability toggles — already built, refit in brand), ADMIN-UX-02 (PATCH toggle — already built, refit in brand), ADMIN-UX-03 (invite form fields + API call — already built, refit in brand), ADMIN-UX-04 (welcome email already sent by backend — no change needed here).
