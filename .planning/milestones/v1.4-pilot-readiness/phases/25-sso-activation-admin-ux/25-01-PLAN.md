Ship the mmctl activation runbook + shell script for Mattermost OIDC SSO.

Purpose: The OIDC bridge code (apps/parrot/workers/routes/oidc.ts, 17 KB) shipped in v1.2. This plan activates the SSO by producing the operator runbook that configures Mattermost to trust the bridge via mmctl.

Mattermost auto-provisioning of users on first OIDC sign-in (MMSSO-03) is native behavior: when Mattermost calls GET /oidc/userinfo with a valid Bearer token, the userinfo endpoint returns { sub, email, email_verified, name, given_name, family_name, preferred_username, username, login, id }. Mattermost's GitLab OAuth module creates a new MM account from these claims if the email does not already exist. No code change is required.

## Secret inventory

Table: four secrets the operator needs and where to find them.

| Env var to set | What it is | Where to get it |
|---|---|---|
| MM_OIDC_CLIENT_ID | Value of MATTERMOST_OIDC_CLIENT_ID | `wrangler secret list --name internjobs-parrot` then `wrangler secret get MATTERMOST_OIDC_CLIENT_ID` |
| MM_OIDC_CLIENT_SECRET | Value of MATTERMOST_OIDC_CLIENT_SECRET | Same — retrieve from Wrangler or Infisical |
| MM_OIDC_REDIRECT_URI | Mattermost callback URL | Should be `https://chat.internjobs.ai/signup/gitlab/complete` |
| MM_SITE_URL | Public Mattermost URL | `https://chat.internjobs.ai` |

Note: These secrets already exist in the Parrot Worker (wrangler secrets). Retrieve them via Wrangler CLI, NOT the CF dashboard (dashboard masks values). If they are missing, generate a new client_id/secret pair (any random 32+ byte hex string) and set them in Wrangler first: `wrangler secret put MATTERMOST_OIDC_CLIENT_ID --name internjobs-parrot`.

## Pre-flight checks

Numbered list:
1. Verify OIDC endpoints are live: `curl -s https://workspace.internjobs.ai/oidc/.well-known/openid-configuration | jq .` — expect JSON with authorization_endpoint, token_endpoint, userinfo_endpoint.
2. Verify mmctl is authenticated: `mmctl auth list` — should show an entry for chat.internjobs.ai.

export MM_OIDC_CLIENT_ID="<value from Wrangler>"
