**DATES-01 scope decision (Advisory #2 resolution):**
The original plan created `packages/shared/src/dates.ts` as a canonical shared home. This is NOT viable: `apps/parrot` and `apps/agentic-inbox` are explicitly excluded from the root npm workspaces (`!apps/agentic-inbox`, `!apps/parrot` in root `package.json`). Neither app can resolve `@internjobs/shared`. Both apps use `baseUrl: "."` in their tsconfig, so `import "shared/dates"` resolves to `./shared/dates.ts` within each app's own directory. Creating `packages/shared/src/dates.ts` would produce an orphan file with zero consumers. True packages/shared consolidation requires workspace wiring surgery — deferred beyond Phase 27 scope.

**Chosen approach (option b):** Drop the `packages/shared/src/dates.ts` creation. Keep the three `@deprecated` re-export deletions (which fully satisfy DATES-01's goal of removing deprecated aliases). Callers already using `"shared/dates"` (the app-local copy) directly need no import-path changes — only the intermediate alias const declarations and any calls to those aliases must be updated.

**Step B — Add `PATCH /api/inbox/messages/:id` route to `apps/parrot/workers/index.ts`:**

Add immediately after the existing `GET /api/inbox/messages/:id` route (around line 353). The DO method `updateEmail(id, {starred})` already exists — this is just the HTTP layer:

```typescript
// STAR-API-01: PATCH /api/inbox/messages/:id — toggle starred/read state.
// Body: { starred?: boolean; read?: boolean }
// The EmployeeMailboxDO.updateEmail() method already exists and handles the
// SQLite UPDATE. This route is the missing HTTP surface the EmailPanel needs.
app.patch(
  "/api/inbox/messages/:id",
  requireEmployeeMailbox,
  async (c: AppContext) => {
    const id = c.req.param("id");
    if (!id) return c.json({ error: "Missing message id" }, 400);
    const body = (await c.req.json().catch(() => null)) as {
      starred?: boolean;
      read?: boolean;
    } | null;
    if (!body || (body.starred === undefined && body.read === undefined)) {
      return c.json({ error: "Body must include starred or read field" }, 400);
    }
    const updated = await c.var.mailboxStub.updateEmail(id, {
      starred: body.starred,
      read: body.read,
    });
    if (!updated) return c.json({ error: "Email not found" }, 404);
    return c.json({ id: updated.id, starred: !!updated.starred, read: !!updated.read });
  },
);
```

```tsx
// In EmailPanel function body, add:
const queryClient = useQueryClient();
const [starred, setStarred] = useState(email.starred ?? false);

async function handleStar() {
  const next = !starred;
  setStarred(next); // optimistic
  try {
    await api.patchMessage(emailId, { starred: next });
    // Invalidate so list view reflects the new starred state on next load.
    queryClient.invalidateQueries({ queryKey: ["parrot", "inbox"] });
  } catch {
    setStarred(!next); // revert on error
  }
}
```
