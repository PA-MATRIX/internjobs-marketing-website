This is a variant of `TodoCard` used in the Resolved view. It must render:
- Same source icon + title layout as `TodoCard`
- A pill badge: violet `bg-violet-100 text-violet-700` for `resolution_source === 'agent'` with text "Agent"; grey `bg-slate-100 text-slate-500` for `'user'` or null with text "You"
- A relative timestamp showing when the todo was resolved ("resolved 2m ago" format — reuse the `formatAge` helper with `resolved_at` instead of `created_at`)
- An inline "Undo" button (only shown when `resolution_source === 'agent'`) — `text-sm text-slate-500 hover:text-slate-700 underline` style, no confirmation dialog

Do NOT merge `ResolvedTodoCard` into the base `TodoCard` with a boolean prop — they have different visual hierarchies and different interaction models.

```tsx
interface ResolvedTodoCardProps {
  todo: TodoItem;
  onUndo: (todo: TodoItem) => void;
}

export function ResolvedTodoCard({ todo, onUndo }: ResolvedTodoCardProps) {
  const { Icon, color, bg } = SOURCE_ICON[todo.source_channel] ?? SOURCE_ICON.email;
  const resolvedAge = todo.resolved_at ? formatAge(todo.resolved_at) : "";

  const pillClass =
    todo.resolution_source === "agent"
      ? "bg-violet-100 text-violet-700"
      : "bg-slate-100 text-slate-500";
  const pillLabel =
    todo.resolution_source === "agent" ? "Agent" : "You";

  return (
    <div className="w-full text-left rounded-xl border border-slate-200 bg-white p-4 shadow-sm flex items-start gap-3 opacity-80">
      <div
        className={`flex-shrink-0 w-9 h-9 rounded-lg flex items-center justify-center ${bg}`}
        aria-label={`${todo.source_channel} source`}
      >
        <Icon size={18} className={color} strokeWidth={2} />
      </div>

      <div className="flex-1 min-w-0">
        <div className="flex items-start gap-2">
          <h3 className="text-sm font-medium text-slate-500 line-clamp-2 flex-1">
            {todo.title}
          </h3>
        </div>

        <div className="mt-2 flex items-center gap-2 flex-wrap">
          <span
            className={`text-[10px] uppercase tracking-wide font-semibold px-1.5 py-0.5 rounded ${pillClass}`}
          >
            {pillLabel}
          </span>
          {resolvedAge && (
            <span className="text-[10px] uppercase tracking-wide font-semibold text-slate-400">
              resolved {resolvedAge}
            </span>
          )}
          {todo.resolution_source === "agent" && (
            <button
              type="button"
              onClick={() => onUndo(todo)}
              className="ml-auto text-xs text-slate-400 hover:text-slate-600 underline"
            >
              Undo
            </button>
          )}
        </div>
      </div>
    </div>
  );
}
```

**`dashboard.tsx` — full updates for animate-out, Resolved view, Undo, and toast.**

The current `DashboardRoute` component fetches todos and renders them via `<TodoCard>`. It needs to support:

1. **`view=resolved` branch**: when `activeView === "resolved"`, fetch `GET /api/dashboard/todos?view=resolved` and render `<ResolvedTodoCard>` items with `onUndo={handleUndo}`.

2. **Animate-out**: track a `dismissingIds: Set<string>` state. When the cron resolves a todo and the parent re-fetches, we don't have a server-push mechanism in v1.3 — instead, use a **polling interval** on the active list: refetch `/api/dashboard/todos` every 10 seconds when `activeView !== "resolved"`. When a previously-visible todo disappears from the response, add its `id` to `dismissingIds` and apply the animate-out CSS for 250ms before the card unmounts.

   CSS for animate-out (inline Tailwind transition):
   ```tsx
   // Applied to the todo's wrapper div while it's in dismissingIds
   className="transition-all duration-[250ms] ease-in-out overflow-hidden"
   style={isDismissing ? { maxHeight: 0, opacity: 0, marginBottom: 0 } : { maxHeight: '200px', opacity: 1 }}
   ```

   To detect a disappearing todo: compare the previous `todos` array IDs to the incoming response IDs. Any ID in `prevIds` that is NOT in `newIds` is a newly-resolved todo — add to `dismissingIds`, wait 250ms, then allow normal re-render.

   Implementation sketch:
   ```typescript
   const prevTodoIdsRef = useRef<Set<string>>(new Set());

   // In the fetch callback, after setState({ status: "ok", todos }):
   const newIds = new Set(todos.map((t) => t.id));
   const disappeared = [...prevTodoIdsRef.current].filter((id) => !newIds.has(id));
   if (disappeared.length > 0) {
     setDismissingIds((prev) => new Set([...prev, ...disappeared]));
     // Show first-auto-clear toast if this is agent-resolution
     // (we can't know which resolution_source disappeared, so check
     //  if any of the current resolved list has resolution_source='agent')
     checkAndShowFirstAgentClearToast(employeeId);
     setTimeout(() => {
       setDismissingIds((prev) => {
         const next = new Set(prev);
         disappeared.forEach((id) => next.delete(id));
         return next;
       });
     }, 300); // slightly longer than 250ms CSS transition
   }
   prevTodoIdsRef.current = newIds;
   ```
