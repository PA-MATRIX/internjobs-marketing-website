# 05-03 Plan: Browser Enrichment Design Gate

## Goal

Decide whether browser-based profile enrichment is safe before building it.

## Must Haves

- Written design for any browser/cloud enrichment.
- Explicit student consent model.
- No LinkedIn credential capture.
- No anti-bot, captcha, or private-page bypass.
- Legal/compliance approval required before production enablement.

## Verification

- Design doc exists and is reviewed before implementation.
- Production code path remains disabled until approval.
