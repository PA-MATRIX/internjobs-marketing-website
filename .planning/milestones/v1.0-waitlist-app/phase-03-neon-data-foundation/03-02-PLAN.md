# 03-02 Plan: Database Schema and Migrations

## Goal

Create a repeatable schema for waitlist identity, consent, messaging, and profile state.

## Must Haves

- Migration tool selected and documented.
- Tables include students, waitlist status, channel pairing codes, consents, profile snapshots, messaging events, and audit events.
- Unique constraints support idempotent writes.
- Timestamps and provider metadata are preserved.

## Verification

- Fresh database migrates from zero.
- Re-running migrations is safe.
- Schema supports repeated Clerk login and repeated webhook delivery.
